## Gaussian Splat Export Investigation

### Problem
Takara's exported gaussian splat PLY files don't render correctly in SuperSplat — appear as random spheroids instead of proper scene. Even the **original untouched PLY** from training doesn't work.

### Root Cause Found
The PLY files from 3DGS training have **62 properties** including:
- 45 spherical harmonics coefficients (`f_rest_0` through `f_rest_44`)
- 3 normal components (`nx`, `ny`, `nz`)
- Standard position, scale, rotation, opacity, f_dc

SuperSplat (and likely other web viewers) expect a **minimal 14-property format**:
- `x, y, z` (position)
- `f_dc_0, f_dc_1, f_dc_2` (base color SH)
- `opacity`
- `scale_0, scale_1, scale_2`
- `rot_0, rot_1, rot_2, rot_3` (quaternion)

### Solution Created
`export_minimal.py` — Converts full 3DGS PLY to minimal viewer-compatible format:
- Strips normals and higher-order SH coefficients
- Normalizes quaternions (original had non-unit quaternions ~0.69-1.02)
- Can sample by opacity for size limits

- gsplat CUDA JIT compilation is broken in this environment (missing `cuda_runtime_api.h` in conda paths)
- The segmentation/transform code works; only local rendering is broken
- Known working reference: `biker.ply` from PlayCanvas GitHub uses the minimal 14-property format

## Pure Mesh Proximity Segmentation (07:00 UTC)

### Breakthrough: Ditched Bboxes Entirely
User pointed out bboxes were causing confusion — switched to **pure collision mesh containment/proximity**.

### Key Findings
1. Only **57 of 237 meshes are watertight** — containment tests fail on most
2. The `ZHXI65RVAIMGCPTULM888888_*` meshes (16 watertight chairs) are NOT near chair 148
3. Chair 148's actual collision mesh: `/Root/Z3LWSQETQVHX6PTULY888888_006` — **not watertight**
4. Table 141's mesh: `/Root/ZTODBAZVAUKPKPTUKM888888` — also not watertight

### Solution: Distance-Based Assignment
Since meshes aren't watertight, use **proximity to mesh surface** instead of containment:
```python
# Find nearest surface point on each mesh
_, chair_dists, _ = chair_mesh.nearest.on_surface(positions)
_, table_dists, _ = table_mesh.nearest.on_surface(positions)

# Assign to chair if: within 3cm AND closer to chair than table
chair_mask = (chair_dists < 0.03) & (chair_dists < table_dists)
```

### Results
- `chair_proximity.ply`: 2,952 gaussians (clean chair, no table contamination)

## Gaussian Splat Rendering + Inpainting Pipeline WORKING (16:00-17:30 UTC)

### Major Breakthrough: Custom 3DGS Rendering
After hours of debugging camera/projection matrix conventions, got `diff_gaussian_rasterization` working:

**The Key Issue:** 3DGS rasterizer expects camera looking down **+Z** (not OpenGL's -Z)
- Build C2W matrix with `forward` in the +Z column (not -forward)
- Transpose W2C for `world_view_transform`
- Camera center = `world_view_transform.inverse()[3, :3]`

**Final Working Code Pattern:**
```python
c2w[:3, 2] = forward  # NOT -forward (camera looks down +Z)
w2c = np.linalg.inv(c2w)
world_view_transform = torch.tensor(w2c.T, device='cuda')  # Transpose!
```

### SuperSplat Compression Issue Found
SAGE InteriorGS uses SuperSplat compressed format:
- **~50% of gaussians have infinite opacity** after decompression
- Must filter: `valid = ~np.isinf(opacity) & ~np.isnan(opacity)`
